USAGE EXAMPLES: CLI AI CODING ASSISTANT
========================================
This document is updated at the end of each completed phase.
Each section shows how to run and verify that phase's functionality.


PHASE 1 — PROJECT SETUP
------------------------
Status: COMPLETE

1. Create and activate virtual environment
   ------------------------------------------
   python -m venv venv

   # Windows
   venv\Scripts\activate

   # Mac/Linux
   source venv/bin/activate

2. Install project dependencies
   ------------------------------------------
   pip install -e .

   Expected output: Successfully installed coding-assistant-0.1.0 and all
   dependencies (langchain, langchain-groq, mcp, rich, typer, chromadb, etc.)

3. Set up environment variables
   ------------------------------------------
   # Copy the example file and fill in your API keys
   cp .env.example .env

   # Open .env and set at minimum:
   GROQ_API_KEY=your_groq_api_key_here
   OPENAI_API_KEY=your_openai_api_key_here

4. Verify config loads correctly
   ------------------------------------------
   python -c "from config import config; print(config)"

   Expected output: Config object printed with provider=groq,
   model=llama-3.3-70b-versatile, max_turns=20, auto_execute=False

5. Verify environment pre-flight checks
   ------------------------------------------
   python -c "from config import config; warnings = config.validate_environment(); print(warnings if warnings else 'All checks passed')"

   Expected output (if keys + tools are set up):
     All checks passed

   Expected output (if a key is missing):
     ["No API key found for provider 'groq'. Set GROQ_API_KEY in your .env file."]

6. Verify Node.js, npx, and Ollama are available
   ------------------------------------------
   node --version    # Expected: v24.x.x or higher
   npx --version     # Expected: 10.x.x or higher
   ollama --version  # Expected: ollama version is 0.x.x

Status: COMPLETE

   Expected output (all checks marked with ✔):
     ✔  AgentLoop, MessageHistory, SYSTEM_PROMPT imported
     ✔  MessageHistory: add_human / add_ai / add_tool_result / clear all pass

     ✔  Tool-calling run completed without error  (list_directory called)

2. Use AgentLoop directly in Python
   ------------------------------------------
   import asyncio
   from config import config
   from providers import get_model_from_config
   from mcp_client import MCPClient, get_server_configs, build_langchain_tools
   from agent import AgentLoop

   async def main():
       model = get_model_from_config(config)
       configs = get_server_configs(config.workspace)
       async with MCPClient(configs) as client:
           tools = build_langchain_tools(client.list_all_tools(), client)
           loop = AgentLoop(model=model, tools=tools, max_turns=20, auto_execute=False)
           await loop.run("Summarise the config.py file")

   asyncio.run(main())

3. Key behaviours
   ------------------------------------------
   Streaming      : LLM tokens display in-place with Rich Live as they arrive
   Tool call      : Yellow panel shows tool name + JSON args before execution
   Confirmation   : y/n prompt per tool call when auto_execute=False
   Tool result    : Green (success) or red (error) panel with truncated output
   Turn warning   : Fires at 75% of max_turns (e.g. turn 15/20)
   Turn limit     : "🛑 Turn Limit Reached" panel stops runaway loops
   History        : Full conversation carried across turns; auto-trims at 80 msgs
   Multi-task     : Call loop.clear_history() between unrelated tasks

4. Controlling the loop
   ------------------------------------------
   # Auto-execute all tools (no prompts)
   loop = AgentLoop(model, tools, max_turns=20, auto_execute=True)

   # Tight budget for cheap exploration tasks
   loop = AgentLoop(model, tools, max_turns=5, auto_execute=True)

   # Reset history between tasks
   await loop.run("Task A")
   loop.clear_history()
   await loop.run("Task B")


PHASE 7 — CLI ENTRY POINT
--------------------------
Status: COMPLETE

Entry point : main.py  (also available as `assistant` after pip install -e .)
All commands below work equivalently with `python main.py` or `assistant`.

1. Show help
   ------------------------------------------
   python main.py --help

   Expected output:
     Usage: main.py [OPTIONS] [TASK]

       Autonomous AI coding assistant — reads, edits, and executes code for you.

     Arguments:
       [TASK]  Task to run immediately (omit for interactive REPL mode).

     Options:
       --auto            Auto-execute tools without asking for confirmation.
       --max-turns INT   Hard limit on agentic loop iterations  [1<=x<=100]
       --provider TEXT   LLM provider: groq | openai | ollama
       --model TEXT      Model name override (e.g. gpt-4o)
       --workspace TEXT  Workspace directory the filesystem server may access
       --setup-rag       Build (or rebuild) the LangChain docs index and exit
       --help            Show this message and exit.

2. Single-task mode (run one task and exit)
   ------------------------------------------
   python main.py "Reply with exactly the word PONG and nothing else."

   Expected output:
     - Welcome banner (provider / model / max-turns / auto-exec)
     - "Connecting to 3 MCP server(s)…" spinner
     - "Connected: ['filesystem', 'context7', 'rag']  (18 tools)"
     - "↳  12 tools active after filtering"
     - Assistant response: PONG
     - "✔  Task complete."
     - "Shutting down MCP servers…" spinner

3. Interactive REPL mode
   ------------------------------------------
   python main.py

   Then type tasks at the ">" prompt.  Built-in commands:
     help   — show command list
     clear  — wipe conversation history (start a fresh context)
     exit   — quit (also: quit, q, :q, or Ctrl+C)

   Example session:
     > List the files in the current directory
     [Tool call panel for list_directory]
     Execute this tool? [y/n]: y
     [Tool result panel with file listing]
     Assistant  The directory contains: agent/, config.py, main.py, ...
     ✔  Task complete.

     > exit
     Goodbye.

4. Auto-execute mode (skip all confirmation prompts)
   ------------------------------------------
   python main.py --auto "List the files in the current directory"

   All tool calls execute immediately without asking y/n.

5. Provider and model overrides
   ------------------------------------------
   # Use OpenAI GPT-4o
   python main.py --provider openai --model gpt-4o "Explain config.py"

   # Use local Ollama (no API key required)
   python main.py --provider ollama --model llama3.2 "List files"

   # Tight turn budget for quick tasks
   python main.py --max-turns 5 "What does main.py do?"

6. Restrict workspace (filesystem server access)
   ------------------------------------------
   # Only allow the agent to read/write inside ./src
   python main.py --workspace ./src "Refactor the login module"

7. Build the RAG index and exit
   ------------------------------------------
   python main.py --setup-rag

   Expected output:
     Setting up LangChain documentation index…
     [same as running python -m mcp_servers.rag_server.indexer]

8. Tool filtering (Groq compatibility)
   ------------------------------------------
   The `@modelcontextprotocol/server-filesystem` (current version) exposes 14
   tools, for a total of 18 across all three servers.  Groq's llama models
   reliably fail tool-call generation above ~12–14 tools ("Failed to call a
   function" error), so main.py filters out 6 redundant/low-value tools:

     Removed                   Reason
     read_text_file            duplicates read_file
     read_media_file           LLM cannot interpret binary media
     read_multiple_files       model can call read_file in sequence
     list_directory_with_sizes duplicates list_directory
     get_file_info             covered by list_directory
     get_rag_status            maintenance tool, not task-relevant

   Result: 12 tools passed to the model (shown as "↳  12 tools active").

9. Error handling
   ------------------------------------------
   - If a provider API call fails, the REPL prints a red error panel and
     returns to the ">" prompt (conversation history is cleared).
   - MCP server startup failures print an error and exit with code 1.
   - Ctrl+C at the ">" prompt exits cleanly after shutting down MCP servers.
   - Ctrl+C during a running task interrupts it and clears history.

10. Known issues resolved
    ------------------------------------------
    - Config.get_api_key_for_provider() takes no arguments: replaced with a
      local _key_map dict lookup in _async_session().
    - Groq "Failed to call a function" crash: fixed by (a) filtering to 12
      tools and (b) passing parallel_tool_calls=False to bind_tools().
